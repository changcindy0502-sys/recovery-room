🚑 恢復室病人運送登記系統 - 雲端快速部署與測試指南本指南提供兩種最適合護理站與科室同仁快速測試的免伺服器部署方案。這兩種方案皆提供免費、安全（HTTPS）且穩定的即時網址，方便您在手機、平板與桌機之間進行多端同步測試。方案一：Vercel 免費雲端部署 (最推薦！只需拖放資料夾)Vercel 提供一個名為 Vercel Deploy 的線上功能，完全不需要安裝任何工具，只要將包含 index.html 的資料夾拖放到瀏覽器中，即可在一瞬間完成上架。步驟 1：準備本地檔案在您的電腦桌面上建立一個新資料夾，命名為 recovery-transport（名稱可自訂）。將您的 index.html 檔案複製並放入該資料夾中。步驟 2：登入 Vercel開啟瀏覽器，前往 Vercel 官方網站 (vercel.com)。註冊或登入帳號（建議直接使用 GitHub、GitLab 帳號登入，或用 E-mail 註冊免費個人帳號 Hobby Account）。步驟 3：使用 Vercel Add New / Import登入後進入 Dashboard（控制台）。前往瀏覽器網址列輸入或點擊這個「拖放專用網址」：vercel.com/import/browse。您會看到一個寫著 "Drag and drop a folder here to deploy it" (拖放資料夾至此部署) 的虛線區塊。將您剛剛準備好的 recovery-transport 資料夾，直接用滑鼠拖放到該虛線區塊中。步驟 4：取得測試網址Vercel 會自動辨識並在 5 ~ 10 秒內編譯完成。畫面上會出現煙火特效，並顯示類似以下格式的專屬測試網址：https://recovery-transport-xxxx.vercel.app點選該網址，就能直接開啟系統！您可以直接複製此網址，用通訊軟體傳給同事，或轉成 QR Code 供護理站同仁掃描。方案二：GitHub Pages 免費靜態網頁託管 (適合長期維護)如果您習慣使用 GitHub 進行程式碼版庫管理，GitHub Pages 是最穩定且完全免費的網頁託管選擇。步驟 1：建立 GitHub 儲存庫 (Repository)登入您的 GitHub 帳號。點擊右上角 「+」 -> 選擇 「New repository」。填寫儲存庫資訊：Repository name (名稱)：例如 recovery-room-transportPublic / Private：請務必選擇 Public (公開)（免費版 GitHub Pages 僅支援公開儲存庫）。點選最下方的 「Create repository」 建立。步驟 2：上傳您的 index.html進入剛建立好的儲存庫頁面，點擊 「uploading an existing file」 連結。將您的 index.html 檔案拖放至瀏覽器畫面中。點選頁面底部的 「Commit changes」 完成上傳。步驟 3：啟用 GitHub Pages 服務在儲存庫頁面頂端，點選 「Settings (設定)」 標籤。在左側選單中找到 「Code and automation」 區塊，點選其中的 「Pages」。在 Build and deployment 下的 Branch 區塊中：將預設的 None 改選為 main (或 master)。資料夾維持選擇 / (root)。點擊右側的 「Save (儲存)」。步驟 4：開啟您的測試網址儲存後稍等約 1 到 2 分鐘，重新整理 Pages 設定頁面。頁面最上方會出現一行綠色的成功橫幅，顯示您的專屬測試網址，格式如下：https://[您的GitHub帳號].github.io/recovery-room-transport/同事點擊該連結即可立刻連線，多人同時使用，數據依然會透過即時資料庫完美同步！💡 護理站同仁分享小撇步行動載具無痕體驗 (PWA)：不論是透過 Vercel 還是 GitHub Pages 產生的網址，都具備安全 HTTPS 憑證。請務必教同事在手機/平板上點選「加入主畫面」，這樣系統開啟時就不會顯示瀏覽器的網址列，操作起來跟醫療專用 App 一模一樣！多裝置即時同步驗證：您可以自己用電腦打開網址，再用手機掃 QR Code 開啟同一個網址。在電腦端新增一筆 POR 床號，您會看見手機端在不重新整理的情況下「瞬間彈出」該筆資料，這代表 Firebase Firestore 雲端即時通道已成功運作！
